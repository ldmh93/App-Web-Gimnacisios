# 8. Roadmap y Funciones Futuras

> Parte de la arquitectura maestra de GymOS SaaS. Índice: [MASTER_ARCHITECTURE.md](./MASTER_ARCHITECTURE.md)

## Futuras Funciones Planeadas

La arquitectura debe permitir agregar:

### Inteligencia Artificial

- Creación automática de rutinas.
- Recomendaciones.
- Análisis de progreso.

### Pagos

- Suscripciones mensuales.
- Planes premium.
- Facturación.

### White Label

Cada gimnasio podría tener:

- Su propio logo.
- Colores.
- Dominio personalizado.

### Aplicación móvil

Preparar backend para futuras apps móviles.

## Roadmap General

### Fase 1 — Conectar Supabase

Crear:

- Usuarios.
- Autenticación.
- Perfiles.
- Roles.

### Fase 2 — Base de datos

Crear:

- Gimnasios.
- Clientes.
- Entrenadores.
- Ejercicios.
- Rutinas.

### Fase 3 — Contenido y seguimiento

Agregar:

- Storage de imágenes.
- Videos.
- Seguimiento físico.

### Fase 4 — Modelo SaaS

- Planes.
- Pagos.
- Administración global.
